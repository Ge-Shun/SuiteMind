生产版安装

在 Word 中安装 SuiteMind

生产版直接从 GitHub Pages 加载。安装后无需运行 Node.js、PowerShell 或 npm run dev。

下载生产版 manifest

文件下载后应命名为 SuiteMind-manifest.xml。

  1. 1

    下载 manifest

    使用上方按钮下载生产版 XML 文件，不要使用项目中的开发版 manifest。

  2. 2

    在 Word 中上传

    打开 Word 网页版，依次选择“插入”→“加载项”→“更多加载项”→“我的加载项”→“上传我的加载项”，然后选择刚下载的 XML 文件。

  3. 3

    打开 SuiteMind

    安装完成后从“我的加载项”打开 SuiteMind。使用同一 Microsoft 账户时，可在支持的 Word 客户端中继续使用。

  4. 4

    配置模型

    在任务窗格的模型配置中填写 Provider、API 地址、API Key 和模型，然后先运行“测试连接”。

看到 localhost 错误或旧图标？

这通常表示电脑里仍保留开发版旁加载项。请先从“我的加载项”移除旧版本，关闭所有 Word 窗口，再安装本页提供的生产版 manifest。

部分模型服务商会阻止 Office 直接连接。此类连接限制与加载项安装无关，请查看项目文档中的 CORS 和本地代理说明。

首次运行可能出现“未知发布者”

SuiteMind 目前没有公开可信的代码签名证书。Windows SmartScreen 显示“Windows 已保护你的电脑”属于预期提示，但下载来源和 SHA-256 校验值必须匹配本页。

  1. 从官方地址下载 只使用本页或 SuiteMind GitHub Releases 提供的 ZIP 文件。
  2. 核对 SHA-256 运行下方命令，并与校验文件中的 64 位哈希值比较。
  3. 解压并运行 解压 ZIP，然后双击 SuiteMindConnector.exe。
  4. 处理 SmartScreen 确认来源和哈希一致后，选择“更多信息”→“仍要运行”。

校验下载文件

在下载目录打开 PowerShell，运行：

Get-FileHash .\SuiteMind-Connector-win-x64.zip -Algorithm SHA256
下载 SHA-256 校验文件

遇到这些情况不要继续

下载域名不是 ge-shun.github.io 或 github.com/Ge-Shun/SuiteMind、哈希值不一致，或者 Windows Defender 明确报告恶意软件、签名无效或文件损坏。

English installation summary

Download the production manifest above. In Word for the web, choose Insert → Add-ins → More Add-ins → My Add-ins → Upload My Add-in, then select the downloaded XML file. Open SuiteMind from My Add-ins and configure your model provider in the task pane. If direct provider access is blocked, download and run the optional Windows connector. The connector is currently unsigned, so verify its SHA-256 checksum before choosing More info → Run anyway in Windows SmartScreen.
